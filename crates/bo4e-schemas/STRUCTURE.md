# STRUCTURE.md — `bo4e-schemas`

The canonical schema layer: typed JSON-Schema models, a closure-based tree walker, version parsing, and disk IO.
Zero workspace dependencies — both `bo4e-codegen` and `bo4e-cli` consume it.

## Purpose

- Represent a BO4E JSON schema as strongly-typed Rust (`SchemaRootType` and friends) with round-trippable serde.
- Group schemas into a versioned `Schemas` collection that maps cleanly to a directory tree.
- Traverse the schema tree with a single object-safe trait (`Visitable`) that downstream crates extend.

## Layout

```
src/
├── lib.rs              # module wiring + re-exports
├── visitable.rs        # `Visitable` trait + visit_all / try_visit_all (mut & immutable)
├── models/
│   ├── mod.rs
│   ├── macros.rs       # literal_enum!, visitable_leaf!, visitable_forwarded_iter!, visitable_dispatch_enum!
│   ├── json_schema.rs  # the typed JSON-Schema enum tree (TypeBase, SchemaRootType, …)
│   ├── schema_meta.rs  # Schema, Schemas (the on-disk model)
│   └── version.rs      # Version, DirtyVersion (incl. `v202401.0.1+gabc.d20240101` shapes)
└── io/
    ├── mod.rs
    └── schemas.rs      # read_schemas / write_schemas + `.version` file handling
```

Re-exports from `lib.rs`: `Schema`, `Schemas`, `Version`, `DirtyVersion`, `Visitable`.

## Key types

### `Schema` and `Schemas` (`models/schema_meta.rs`)

- `Schema` carries a `module: Vec<String>` (path segments from the schemas dir, last segment = name) and a lazily parsed `SchemaRootType`. The raw `_schema_text` is kept around for cheap round-trip serialisation when nothing was mutated.
- `Schemas` is a versioned collection (`HashMap<String, Rc<RefCell<Schema>>>` keyed by name, plus a `version: DirtyVersion`). It implements `IntoIterator`.
- Schemas are wrapped in `Rc<RefCell<…>>` because the edit/diff layers need to mutate individual schemas while keeping the collection iterable.

### `SchemaRootType` and the JSON-Schema enum tree (`models/json_schema.rs`)

A discriminated union of every JSON-Schema shape this codebase cares about: objects, arrays, `allOf` / `anyOf`, references, primitives, string enums. Each variant has a `TypeBase` (description / title / default).

- `PrimitiveValue` uses an explicit `deserialize_present_primitive` helper so an *explicit* JSON `null` becomes `Some(PrimitiveValue::Null)`, not `None` (serde's default would silently drop a `"default": null`).
- `$defs` and `$definitions` are both accepted on input via serde `alias`.
- The `Visitable` impls for every variant are generated by the macros in `models/macros.rs`.

### `Version` and `DirtyVersion` (`models/version.rs`)

- `Version` parses `v202401.0.1` and `v202401.0.1-rc2`.
- `DirtyVersion` adds optional `+g<commit>` and `.d<YYYYMMDD>` suffixes. **Important**: the two suffixes are independent — `bo4e edit` brands its output with `.d<date>` even when the input was a tagged release (no commit hash), so the date suffix must parse on its own.
- Both implement `Display` and `FromStr`; `Version: From<&str>` returns errors as `String`.

### `Visitable` (`visitable.rs`)

```rust
pub trait Visitable: Any + Debug {
    fn for_each_child(&self, f: &mut dyn FnMut(&dyn Visitable));
    fn for_each_child_mut(&mut self, f: &mut dyn FnMut(&mut dyn Visitable));
}
```

Closure-based on purpose: it's object-safe and avoids `RefCell`-style runtime borrow checks for mutable traversal. `dyn Visitable` gets four blanket helpers:

- `visit_all::<T>(&mut FnMut(&T))` and `visit_all_mut::<T>(&mut FnMut(&mut T))` for unconditional walks.
- `try_visit_all::<T, B>` / `try_visit_all_mut::<T, B>` that take a closure returning `ControlFlow<B>` for early termination. The `try_*` variants are how the `diff` and `edit` modules abort traversal on the first match.
- Order: depth-first pre-order (self before children).
- Helpers `cntrl_to_result` / `result_to_cntrl` bridge `ControlFlow` and `Result`.

The macros in `models/macros.rs` keep `Visitable` impls one-liners:

- `visitable_leaf!(T)` — type has no schema children.
- `visitable_forwarded_iter!(T, field)` — children are `self.field`'s items.
- `visitable_dispatch_enum!(E, V1, V2, …)` — dispatches to the inner variant's `Visitable` impl.
- `literal_enum!(N, Variant)` — single-variant serde enum used for `"type": "object"` markers etc.

## IO

`io::schemas::read_schemas(input_dir)` walks `input_dir` with `walkdir`, skipping dotfiles, reads every `*.json`, and assembles a `Schemas` keyed by basename. The relative path (minus extension) becomes the `module`. The `.version` file at the root carries the upstream version.

`io::schemas::write_schemas(&schemas, output_dir)` does the inverse: creates each `module`'s parent dirs and writes a `.version` file with the schemas' `DirtyVersion`.

`ReadSchemasOutput.warnings` collects non-fatal traversal issues (e.g. unreadable entries) without failing the whole pull/edit.

## Conventions

- Schema names are deduplicated by basename — two schemas with the same final module segment is a hard error in `Schemas::add_schema`.
- `Rc<RefCell<Schema>>` is the public handle. Borrow scope tightly (the `edit` transforms drop borrows before re-entering the tree).
- Don't expose `serde_json::Value` from this crate's public API — promote the JSON-Schema shape into the typed tree first.
- New schema variants:
  1. Add a struct/enum in `models/json_schema.rs`.
  2. Add it to whichever `SchemaRootType` / `SchemaType` / `SchemaClassType` union covers it.
  3. Use the `visitable_*` macros to implement `Visitable` so traversal stays exhaustive.
  4. Update the `diff` walker in `bo4e-cli/src/diff/diff.rs` to compare the new shape.

## Tests

Unit tests are inline (`#[cfg(test)]`). Notable suites: the visitable test tree in `visitable.rs`, the `read_schemas` fixture builder in `io/schemas.rs`, and round-trip serialisation tests for primitives in `models/json_schema.rs`. `tempfile` is used for filesystem fixtures.
